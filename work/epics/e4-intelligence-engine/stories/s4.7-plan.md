**Rationale (risk-first):** Task 1 first because it unblocks both retrofits. Task 2 (switcher) is fully independent and can be done in parallel conceptually, but sequential execution is cleaner. The form retrofit (Task 3) is the highest-risk change (most files, most strings, tests need updating) вЂ” placing it early maximizes feedback time. The page retrofit (Task 4) is trivial (4 strings). Integration test last (Task 5).

## Risks & Mitigations

| Risk | Mitigation |
|------|------------|
| `useFormatter()` from next-intl needs `NextIntlClientProvider` in test wrapper | `renderWithIntl` already wraps with `NextIntlClientProvider` вЂ” should work. If not, add formatter to the provider |
| Server component `getTranslations()` needs `next-intl` plugin config | `next-intl` v4 with `[locale]` routing already configured. `getTranslations()` should work in async server components without extra config |
| Test assertions use hardcoded Spanish text that may change with i18n keys | Message keys use the same Spanish values as the original hardcoded strings, so existing tests pass without modification. New tests added for locale-specific behavior |
