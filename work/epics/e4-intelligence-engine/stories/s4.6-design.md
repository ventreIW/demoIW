# Design: s4.6 — Rescore-on-Contact Endpoint

**Story ID:** s4.6 · **Epic:** E4 · **Owner:** Nano · **Branch:** `story/s4.6/rescore-on-contact`

---

## Problem

E5 operations panel needs to update a client's score in real-time after a collector records a contact result (promise_to_pay, refusal, partial_payment, etc.). Currently no endpoint exists to trigger a re-score based on contact outcome.

## Value

Collections operators can record contact results and immediately see the updated queue priority without waiting for a full re-score of the entire portfolio. This enables real-time queue management.

---

## Approach

Add `POST /api/v1/scenarios/{scenario_id}/clients/{client_id}/rescore` endpoint that:
1. Accepts a contact result (promise_to_pay, refusal, partial_payment, no_answer, wrong_number)
2. Adjusts the client's score using a simple deterministic heuristic
3. Re-ranks the portfolio using the existing `PrioritizeScenario` use case
4. Returns the updated `PrioritizedPortfolio` with the new score and re-computed Pareto subset

## Design Decisions

### D1 — Score adjustment heuristic

**Rationale:** Demo uses simple deterministic heuristic. Not calibrated — explicitly a demo heuristic.

| Contact Result | Delta |
|----------------|-------|
| `promise_to_pay` | +10 |
| `partial_payment` | +5 |
| `no_answer` | 0 |
| `wrong_number` | 0 |
| `refusal` | -15 |

After adjustment, clamp score to `[0, 100]`.

### D2 — Re-use PrioritizeScenario for re-ranking

The existing `PrioritizeScenario` implements ranking + Pareto subset. Rescore endpoint fetches current scores, adjusts one client's score, calls `PrioritizeScenario` with modified scores.

### D3 — Rescore is ephemeral (no persistence)

Score persistence is s4.9. This story only computes and returns updated prioritization. No `IScoreRepository` usage.

### D4 — Contact result enum

Re-uses existing `ContactResultType` enum: `promise_to_pay`, `partial_payment`, `refusal`, `no_answer`, `wrong_number`.

## Acceptance Criteria

| ID | Description | Severity |
|----|-------------|----------|
| AC1 | POST returns 200 with updated `PrioritizedPortfolio` | must |
| AC2 | Invalid `contact_result` → 422 with valid options | must |
| AC3 | Unknown `scenario_id` → 404 | must |
| AC4 | Unknown `client_id` → 404 | must |
| AC5 | Unscored scenario → 404 | must |
| AC6 | `promise_to_pay` → score +10 (clamped to 100) | must |
| AC7 | `refusal` → score -15 (clamped to 0) | must |
| AC8 | `partial_payment` → score +5 | must |
| AC9 | `no_answer` / `wrong_number` → score unchanged | must |
| AC10 | Response includes updated Pareto subset | must |
