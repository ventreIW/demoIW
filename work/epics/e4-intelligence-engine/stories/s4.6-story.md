# Story: Rescore-on-Contact Endpoint

**Owner:** Nano · **Branch:** `story/s4.6/rescore-on-contact`
**Prerequisite:** s4.3 merged (✅ done)

## User Story

As a **collections operator**,
I want a rescore endpoint that updates a client's score when a contact result is recorded,
so that the queue reflects the latest information without waiting for a full re-score.

## Acceptance Criteria

### Scenario: Rescore endpoint updates score given contact result

```gherkin
Given an active scenario with scored clients
And a client with current score 45
When POST /api/v1/scenarios/{scenario_id}/clients/{client_id}/rescore with contact_result "promise_to_pay"
Then the response contains the updated score for that client
And the score is different from the original
And the queue (prioritized list) reflects the new score
```

### Scenario: Rescore endpoint returns 404 for unknown client

```gherkin
Given an active scenario
When POST /api/v1/scenarios/{scenario_id}/clients/{unknown_client_id}/rescore
Then the response is 404 with clear message
```

### Scenario: Rescore endpoint returns 404 for unscored scenario

```gherkin
Given a scenario with no scores
When POST /api/v1/scenarios/{scenario_id}/clients/{client_id}/rescore
Then the response is 404 with clear message
```

## Examples (Specification by Example)

| Input | Action | Expected Output |
|-------|--------|-----------------|
| Client score 45, contact "promise_to_pay" | Rescore | Updated score > 45 (positive signal) |
| Client score 70, contact "refusal" | Rescore | Updated score < 70 (negative signal) |
| Client score 50, contact "no_answer" | Rescore | Score unchanged (neutral) |

## Notes

- This endpoint is consumed by E5 (operations panel) for real-time queue updates after collector contacts
- The rescore should be fast — it re-ranks the portfolio without re-fitting the model
- Uses `PrioritizeScenario` use case from s4.5-API internally
- Score persistence is s4.9 (separate story) — this story only computes and returns the updated score
- Contact result types: "promise_to_pay", "partial_payment", "refusal", "no_answer", "wrong_number"
