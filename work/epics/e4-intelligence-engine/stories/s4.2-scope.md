# s4.2 Scope ‚Äî Feature Engineering & Training Set

**Owner:** Rodrigo ¬∑ **Size:** S ¬∑ **Branch:** `story/s4.2/feature-engineering`

## In Scope

- `ClientFeatures` value object ‚Äî the feature row schema
- `FeatureExtractor` service: `RawDataset` ‚Üí client-level feature frame
- `OutcomeLabeller` service: 90-day forward simulation per client, seeded
- `TrainingSet` value object: `X`, `y`, client ids, split metadata
- `BuildTrainingSet` use case wiring extraction + labelling + split
- Split **by client**, with both classes present on each side
- **Leakage test** asserting `payment_history_pattern` is absent from features
- Reproducibility test: same seed ‚Üí identical features and labels
- Class-balance sanity test: `on_time` cohort collects materially more than `default`
- Clear domain error when only one label class is present

## Out of Scope

| Item | Reason |
|---|---|
| Model training, scoring, evaluation | s4.3 |
| Score explanation | s4.4 |
| Score persistence (`IScoreRepository`) | s4.9 (Nano) |
| Any HTTP endpoint | Nothing consumes this yet; s4.3 does |
| Feature scaling / normalization | Belongs with the model, s4.3 |
| Feature selection experiments | YAGNI ‚Äî start with the design's feature list, revisit if s4.3 underperforms |
| Persisting the training set | Built on demand per ADR-007; no artifact storage |

## Done when

- [ ] `FeatureExtractor` returns one row per client from a `RawDataset`
- [ ] Features are exactly those in `design.md` ¬ß Key contracts
- [ ] A test asserts `payment_history_pattern` is **not** among the feature columns
- [ ] `OutcomeLabeller` produces `collected_within_90d ‚àà {0,1}` per client using `_PATTERN_PROFILES`
- [ ] Labeller imports the profiles from the generator ‚Äî constants are not duplicated
- [ ] Same scenario seed produces identical features **and** identical labels across runs
- [ ] `on_time` clients show a materially higher collection rate than `default` clients
- [ ] Split is by client; no client id in both partitions; both classes present in each
- [ ] Single-class training set raises a clear domain error instead of returning silently
- [ ] Clients with no open invoices produce zeros, not NaN
- [ ] `pytest` green, `ruff check`, `ruff format --check`, `mypy app/` clean
- [ ] ADR-006 referenced from the code that enforces it

## Risks

| Risk | Mitigation |
|---|---|
| The 90-day horizon produces a badly skewed class balance | Sanity test reports the balance; if it is extreme, revisit the horizon in ADR-006 before s4.3 rather than after |
| Coupling the labeller to `_PATTERN_PROFILES` ties E4 to E3 internals | Deliberate and documented in ADR-006 ‚Äî one source of truth for the causal model beats a silently divergent copy |
| Features look fine but carry no signal | s4.3's baseline comparison is the real check; M1 is an explicit go/no-go |
