# s4.4 Scope — Score Explanation

**Owner:** Rodrigo · **Size:** S · **Branch:** `story/s4.4/score-explanation`

## In Scope

- `ScoreExplainer` service: feature contributions → ranked, human-readable Spanish text
- **Grouping of correlated features** so overdue ageing is reported once, not twice
- Materiality filter: negligible contributors are omitted rather than padded to a fixed count
- Phrasing that names the *business fact* (`78 días de atraso`) rather than the feature
  (`days_overdue_max`)
- `ScoreScenario` wired to populate `Score.explanation` with real text, replacing s4.3's
  placeholder
- Deterministic output — same seed, same explanation

## Out of Scope

| Item | Reason |
|---|---|
| LLM-generated explanation prose | Must work without `OPENROUTER_API_KEY`, which is still unavailable. Also non-deterministic |
| English translation of explanations | E7's translation pass; ES-first per project constraints |
| UI rendering of explanations | E5's case detail view |
| Changing the model or its features | s4.3 is merged and its gate passed; this story reads contributions, it does not alter them |
| Counterfactuals ("pay X and the score rises to Y") | Genuinely useful, genuinely a different story. Parking lot |
| Score persistence | s4.9 (Nano) |

## Done when

- [ ] Every `Score` carries an explanation naming its top drivers in Spanish
- [ ] Explanations contain no feature names, coefficients, or statistical vocabulary
- [ ] `days_overdue_max` and `days_overdue_mean` produce **one** ageing statement, not two
- [ ] A low-scored client's explanation names predominantly negative contributors; a high-scored
      client's names positive ones
- [ ] Negligible contributors are omitted — no fixed-count padding
- [ ] Same scenario seed produces byte-identical explanations
- [ ] `Score.explanation` is never empty
- [ ] `pytest` green; `ruff check`, `ruff format --check`, `mypy app/` clean
- [ ] No `sklearn` import outside `adapters/scoring/` — **asserted by a test**, not just stated
      (s4.3's AC #15 was written down and violated within the same story)

## Risks

| Risk | Mitigation |
|---|---|
| Explanation contradicts the score — the failure s4.3's canary exposed | Test asserts the sign of named factors matches the score's direction. Signs are stable at `C=0.01`, verified across 45 configurations |
| Grouping hides a genuinely distinct signal | Group only the pair measured at 0.872 correlation; do not invent broader groupings without evidence |
| Template text reads as robotic in a demo | Phrase around business facts, and review the actual output on a real scenario rather than judging from unit tests |
| Materiality threshold chosen arbitrarily | Derive it from the observed contribution distribution, as s4.3 did for category thresholds |
