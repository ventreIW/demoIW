## The defect reading found

T2's first version passed all 27 unit tests and produced this on real data:

```
[78.9 high] Buen perfil de pago: pagos parciales pendientes.
[86.5 high] Buen perfil de pago: facturas con hasta 40 días de atraso, ...
```

The ranking was correct — a *positive* ageing contribution means the client is less overdue than
the portfolio average — but the sentence rendered the bare fact as though it were a problem,
under a headline calling the client good.

**No unit test would have caught this.** Every rule held; the output still contradicted itself.
This is what the plan meant by "T3 is where the story is judged", and it surfaced a task early.

Fix: phrasing follows the sign of the contribution, not just its value.

| Factor | Favourable | Unfavourable |
|---|---|---|
| Ageing | `atraso contenido, de hasta 40 días` | `facturas con hasta 480 días de atraso` |
| Partials | `pagos parciales recibidos` | `pagos parciales pendientes` |
| Settlement | `5 de 7 facturas liquidadas` | `sólo 2 de 7 facturas liquidadas` |
| Outstanding | `saldo bajo, de $4,000` | `$86,939 pendientes de cobro` |

Three regression tests pin it, each naming the sentence that motivated it.

## The materiality threshold is doing work

Clause counts are spread rather than pinned at the cap, across all three sectors. If every
client landed on exactly 3 the threshold would be inert; if every client landed on 1 it would be
throttling real signal. Asserted by `test_factor_count_distribution_is_not_degenerate`.

## Second test-quality lesson, same as s4.3's

The AC #12 boundary test was written first as a **substring grep** for `"sklearn"`. It failed
immediately on two false positives: a docstring *describing* the rule, and a module importing our
own `SklearnScorer` class. Rewritten to parse imports with `ast`.

That is precisely the mistake s4.3's retrospective recorded — *"tests that read documentation
instead of behaviour fail for the wrong reasons"* — repeated one story later by the person who
wrote it down. Knowing a lesson is not the same as applying it.

The detector was then verified non-vacuous: lifting the `adapters/scoring/` exclusion makes it
report exactly the two files that legitimately import sklearn.
