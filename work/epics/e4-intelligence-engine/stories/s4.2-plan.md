## Task count note

Guidance for an S story is 2‚Äì3 tasks. This plan has **4 implementation tasks + 1 mandatory
integration task**. The deviation is deliberate: the components are independently testable and
the project rule is one commit per logical change. Merging T2 and T3 would bundle the leakage
guard with the labelling algorithm ‚Äî two different risks in one commit.

**Rationale.** T1 first because the 16 existing generator tests are the safety net for the move.

order or together. T4 needs both. T5 last, on real data.

## Risks

| Risk | Mitigation |
|---|---|
| T3's cohort separation fails ‚Äî the label carries no signal | Fail here, not in s4.3. Revisit D2's horizon in ADR-006 before building the model |
| T1 changes generator behaviour subtly | The 16 existing generator tests run as part of T1's verification, before anything depends on the move |
| Class balance is worse than D3 predicts (e.g. 95/5) | T5 asserts a plausible band and records the actual figure. If it is extreme, s4.3's baseline and metric choice must change ‚Äî better known now |
| Over-decomposition for an S story | Accepted deliberately; see the note at the top |
