## Tasks

### T1 — `ScoreExplainer`: grouping, ranking, materiality

The decision logic. Which factors survive, in what order — no prose yet beyond a factor label.

**Files**
- create `backend/app/domain/value_objects/explanation_factor.py` — a ranked factor
- create `backend/app/application/services/score_explainer.py`
- create `backend/tests/unit/test_score_explainer.py`

**TDD**
- **RED** — `test_ageing_reported_once` (D2: `days_overdue_max` + `days_overdue_mean` collapse to
  one factor whose contribution is their sum — the measured 26.2% case);
  `test_grouping_happens_before_ranking` (**the subtle one**: a client where ageing wins only
  *after* summing must show ageing as its top factor);
  `test_factors_below_threshold_omitted` (D1, 30%);
  `test_at_most_three_factors`; `test_dominant_factor_yields_single_reason`;
  `test_direction_matches_score`
- **GREEN** — pure functions over the contributions dict
- **REFACTOR** — threshold and cap as named constants carrying the measured decay that justifies them

**Verification**
```
rai gate check gate-tests --scope backend/tests/unit/test_score_explainer.py
rai gate check gate-lint && rai gate check gate-types
```

**Size:** S · **Depends on:** — · **AC:** #1, #2, #3, #4

> `test_grouping_happens_before_ranking` is the task's real content. Collapsing ageing at render
> time instead would let it lose a slot it should have won — the same split-weight problem that
> destabilised s4.3's coefficients, in presentation form.

## Execution order

```
T1 (group + rank + threshold) ──→ T2 (phrasing + wiring) ──→ T3 (integration + read output)
```

Strictly sequential — each task consumes the previous one's output. **Acyclic.**

Risk-first does not reorder anything here: T1 carries the only real logic risk and is already
first by dependency.

## Risks

| Risk | Mitigation |
|---|---|
| Grouping applied after ranking, so ageing loses a slot it should win | `test_grouping_happens_before_ranking` constructs exactly that case |
| Explanation contradicts the score — the s4.3-class failure | `test_direction_matches_score`, plus stable signs from ADR-007 Amendment 1 |
| Text is technically correct and reads as robotic | T3 prints real output for human reading; unit tests cannot catch register |
| 30% threshold tuned on retail misbehaves elsewhere | T3 samples all three sectors and asserts the factor-count distribution is not degenerate |
| `_placeholder_explanation` survives beside the new path | `test_placeholder_is_gone` in T2 — the legacy sweep as an assertion, not an intention |
