# s4.5-API Scope — Prioritized List Endpoint

**Owner:** Nano · **Size:** S · **Branch:** `story/s4.5/prioritized-list-endpoint`
**Prerequisite:** s4.5-formula contract agreed (T1 in 5d8980f)

## In Scope

- `GET /api/v1/scenarios/{scenario_id}/prioritized` endpoint
- Wiring `PrioritizeScenario` use case to HTTP layer
- Query parameters:
  - `threshold` (float, default 0.80) — Pareto threshold override
  - `sort` (str) — one of: `rank`, `score`, `outstanding`, `expected_recoverable`, `days_overdue`
  - `order` (str) — `asc` | `desc`
  - `category` (str) — filter by `High` | `Medium` | `Low`
  - `days_overdue_min` (int) — filter by minimum days overdue
- Response model matching `PrioritizedPortfolio` contract exactly
- 404 when scenario has no scores
- OpenAPI documentation with examples

## Out of Scope

| Item | Reason |
|------|--------|
| Formula logic (priority value, Pareto) | **s4.5-formula (Rodrigo)** — this story only exposes it |
| Score persistence (`IScoreRepository`) | s4.9 (Nano) — separate story |
| Rescore-on-contact endpoint | s4.6 (Nano) |
| Operations panel UI consuming the endpoint | E5 s5.1 |
| Collector capacity / workload balancing | Parking lot |

## Done when

- [ ] `GET /api/v1/scenarios/{scenario_id}/prioritized` returns 200 with correct contract
- [ ] `threshold` query param overrides default 80%
- [ ] `sort` + `order` params sort the full portfolio (not just subset)
- [ ] `category` filter works and Pareto is computed on filtered set
- [ ] `days_overdue_min` filter works
- [ ] 404 for unscored scenario with clear message
- [ ] Response fields match `PrioritizedCase` / `PrioritizedPortfolio` exactly
- [ ] `pytest` green; `ruff check`, `ruff format --check`, `mypy app/` clean
- [ ] No unused public API or parameters (AR gate)

## Risks

| Risk | Mitigation |
|------|------------|
| Contract drift from s4.5-formula | Build against the value objects in commit 5d8980f; any change = contract change |
| Sorting the subset instead of the full portfolio | The Pareto subset is a view on the full ranking; sort the full list, then take prefix |
| Filtering changes the Pareto threshold meaning | Re-compute Pareto on the filtered set; document this in OpenAPI |
| Frontend expects different field names | The value object IS the contract — don't rename; update frontend if needed |
