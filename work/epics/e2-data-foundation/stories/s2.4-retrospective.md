## Heutagogical Checkpoint

1. **What did I learn?**
   - Vitest `vi.mock` hoisting: factory functions cannot reference module-level variables because `vi.mock` is hoisted to the top. Always use `vi.fn()` inline in the factory, then `import` the mocked function and cast it to get a mutable reference.
   - Next.js server/client boundary: server components can import `"use client"` components directly — no `dynamic()` wrapper needed. The client wrapper pattern (`CsvUploadWrapper`) is the cleanest way to inject `useRouter` into a server component's tree.

2. **What would I change about the process?**
   - Add a "team assignment" check during story design for multi-person projects. The scope.md already said "frontend only" — I should have trusted that over the epic scope which listed backend items. When they conflict, ask the user.

3. **Framework improvements?**
   - The RaiSE process handled the mid-stream correction well: plan updated, design updated, commits structured. No process gaps.
   - The WSL vitest patterns reference (`nextjs-vitest-patterns.md`) was very helpful — the hoisting fix and `tsc` direct path are documented there. Would have saved 2 minutes if loaded earlier.

4. **What am I more capable of now?**
   - Confident with Next.js App Router file upload patterns: `FormData`, client component with hidden file input, server component integration via client wrapper.
   - Vitest mock patterns for Next.js: `useRouter` mock, module mock with hoisting-safe factories, `getByLabelText` for hidden inputs.
   - RaiSE story lifecycle: design → plan → implement → review flow is muscle memory now.

## Patterns added / reinforced

- **File upload in Next.js App Router**: Client component with hidden `<input type="file">`, `FormData` + `fetch` for multipart upload, client wrapper for `useRouter` injection. Pattern: server page → client wrapper (`useRouter`) → client component (file input + API call).
- **Vitest mock hoisting-safe pattern**: `vi.mock("module", () => ({ fn: vi.fn() }))` → `import { fn } from "module"` → `const mockFn = fn as ReturnType<typeof vi.fn>`. Never reference module-level variables inside `vi.mock()` factories.
