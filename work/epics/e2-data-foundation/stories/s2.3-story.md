# Story: s2.3 — Scenario Selector UI

## User Story
As a demo facilitator,
I want to see all available scenarios on the entry-point screen and activate one with a single click,
so that the demo begins from a consistent, known dataset.

## Acceptance Criteria

### Scenario: List all scenarios on entry
```gherkin
Given the database has 3 seed scenarios (Caso 1, Caso 2, Caso 3)
When the user navigates to /scenarios
Then all 3 scenarios are displayed as cards
And each card shows: name, sector badge, status badge, and client count
```

### Scenario: Activate a scenario
```gherkin
Given the user is on /scenarios with multiple scenarios listed
And no scenario is currently active
When the user clicks "Seleccionar" on "Caso 1"
Then a loading state is shown on the card during the API call
And "Caso 1" card updates to show "Activo" badge
And other cards remain unchanged (no full page reload)
```

### Scenario: Active scenario is visually distinguished
```gherkin
Given "Caso 1" is the active scenario
When the user views /scenarios
Then "Caso 1" card has a highlighted border (ring-2 ring-primary)
And "Caso 1" card shows "Activo" badge instead of "Seleccionar" button
And non-active cards show the "Seleccionar" button
```

### Scenario: Route guard redirects when no scenario is active
```gherkin
Given no scenario is active
When the user navigates to / or /operations
Then they are redirected to /scenarios
```

### Scenario: Route guard allows navigation when scenario is active
```gherkin
Given "Caso 1" is the active scenario
When the user navigates to / or /operations
Then they reach the destination (no redirect to /scenarios)
```

### Scenario: Stub buttons are visible but disabled
```gherkin
Given the user is on /scenarios
Then "Generar nuevo" button is visible but disabled
And "Subir CSV" button is visible but disabled
```

### Scenario: Responsive layout
```gherkin
Given the user is on /scenarios on a mobile viewport (< 640px)
Then scenario cards stack vertically in a single column
```

## Examples (Specification by Example)

| Input | Action | Expected Output |
|-------|--------|-----------------|
| GET /api/v1/scenarios returns 3 scenarios | Page renders | 3 ScenarioCards in a responsive grid |
| Click "Seleccionar" on scenario id=1 | PATCH /api/v1/scenarios/1/activate | Card 1 shows "Activo" badge, ring-2 border |
| Cookie `active_scenario_id` is missing | Navigate to `/operations` | HTTP 307 redirect to `/scenarios` |
| Cookie `active_scenario_id=1` is set | Navigate to `/operations` | Page loads normally |
| Window width = 375px | Grid renders | Cards in single column (grid-cols-1) |
| `client_count === 0` | Badge renders | "Sin datos" badge (gray) |
| `client_count > 0` | Badge renders | "Con datos" badge (blue) |
| `status === "active"` | Badge renders | "Activo" badge (green) |

## Notes
- Fetches initial scenario list via server component with `cache: 'no-store'`
- Activation via client-side `fetch` + `router.refresh()` to re-run server component
- Active scenario ID stored in a cookie (`active_scenario_id`) set by the API client after successful activation
- Backend URL configured via `NEXT_PUBLIC_API_URL` environment variable
- Spanish strings for UI text (i18n is B-15, out of scope)
