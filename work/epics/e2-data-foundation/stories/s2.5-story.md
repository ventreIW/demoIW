# Story: CSV Upload Backend

## User Story
As a demo facilitator,
I want to upload a CSV file to create a scenario with clients and invoices,
so that I can populate scenarios from external data without manual API calls.

## Acceptance Criteria

### Scenario: Upload valid CSV creates scenario with clients and invoices
```gherkin
Given a valid CSV file with columns "client_name", "amount", "due_date", "invoice_id"
When I POST the file to /api/v1/scenarios/upload-csv
Then a new Scenario is created with clients and invoices from the CSV
And the response is 201 with ScenarioSummary
```

### Scenario: Upload CSV with missing required columns
```gherkin
Given a CSV file missing the "client_name" column
When I POST the file to /api/v1/scenarios/upload-csv
Then the response is 422
And body.detail contains an error message listing the missing columns
```

### Scenario: Upload empty file
```gherkin
Given an empty file
When I POST it to /api/v1/scenarios/upload-csv
Then the response is 422
And body.detail contains "Empty file"
```

### Scenario: Upload non-CSV file
```gherkin
Given a text file that is not valid CSV
When I POST it as a .csv file to /api/v1/scenarios/upload-csv
Then the response is 422
And body.detail contains a parsing error message
```

## Examples (Specification by Example)

| Input | Action | Expected Output |
|-------|--------|-----------------|
| CSV with headers: client_name,amount,due_date,invoice_id + 3 rows | POST upload-csv | 201, ScenarioSummary with name from filename, sector=null, client_count=3 |
| CSV missing "amount" column | POST upload-csv | 422, detail=[{msg:"Missing required columns: amount"}] |
| Empty file (0 bytes) | POST upload-csv | 422, detail=[{msg:"Empty file"}] |
| File with .txt extension, not valid CSV | POST upload-csv | 422, detail contains parse error |

## Notes
- Backend endpoint missing from s2.4 — frontend already built against this contract.
- Scenario name: derived from uploaded filename (without extension).
- Sector: null/default — user can update later.
- CSV format documented in README.md per E2 design.
- Uses existing IScenarioRepository + entity pattern from s2.2.
