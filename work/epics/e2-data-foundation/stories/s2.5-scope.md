# S2.5 Scope — CSV Upload Backend

## In Scope
- `POST /api/v1/scenarios/upload-csv` endpoint accepting `multipart/form-data` with a CSV file
- CSV parsing with `csv` stdlib module — no external dependencies
- Required column validation: `client_name`, `amount`, `due_date`, `invoice_id`
- 422 response with FastAPI-standard `detail: [{msg: string}]` on validation/parse errors
- On success: create Scenario + Client + Invoice rows, return ScenarioSummary (201)
- Integration tests for: valid CSV, missing columns, empty file, malformed CSV
- Scenario name derived from uploaded filename (without extension)

## Out of Scope
- Frontend changes (already done in s2.4)
- CSV generation / export
- Updating existing scenarios via CSV
- Client deduplication logic (simple insert for now)
- Sector extraction from CSV (set to null/default)
- Large file handling (>10MB) — can be added in a follow-up

## Done when
- [ ] `POST /api/v1/scenarios/upload-csv` returns 201 for valid CSV
- [ ] `POST /api/v1/scenarios/upload-csv` returns 422 for missing columns
- [ ] `POST /api/v1/scenarios/upload-csv` returns 422 for empty file
- [ ] `POST /api/v1/scenarios/upload-csv` returns 422 for malformed CSV
- [ ] Integration tests cover all 4 scenarios above
- [ ] `mypy app/` passes
- [ ] All backend tests pass (`uv run pytest`)
- [ ] Frontend CSV upload flow works end-to-end (manual smoke test)
