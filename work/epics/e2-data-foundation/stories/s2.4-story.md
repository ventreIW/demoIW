# s2.4 — CSV Upload

**Epic:** E2 — Data Foundation  
**Branch:** `s2.4-csv-upload` (from `e2-data-foundation`)  
**Size:** XS  
**Owner:** —  
**Depends on:** s2.2 (can run in parallel with s2.3)

---

## User story

> As a demo facilitator, I want to upload a client CSV file to create a custom scenario so that prospects can see their own data structure reflected in the demo.

---

## Acceptance criteria

- [ ] `POST /api/scenarios/upload-csv` with a valid CSV (multipart/form-data) returns 201 + ScenarioSummary
- [ ] The created scenario appears in `GET /api/scenarios`
- [ ] Required CSV columns: `client_name`, `invoice_id`, `amount`, `due_date` — missing column returns 422 with specific field error
- [ ] Extra columns are ignored without error
- [ ] Invalid `due_date` format returns 422 with row number and column name
- [ ] CSV with zero data rows (header only) returns 422
- [ ] File size > 5 MB returns 413
- [ ] Frontend: "Subir CSV" button on scenario selector opens a file picker (`.csv` only)
- [ ] Upload progress is shown; success navigates to the selector with new scenario visible
- [ ] Integration tests cover: valid upload, missing column, bad date, empty file, oversized file
