# Story: S2.4: CSV Upload Frontend

## User Story
As a user,
I want to upload a CSV file to define a scenario via the scenario selector,
so that I can quickly create scenarios with column mapping and validation.

## Acceptance Criteria

### Scenario: Successful CSV upload with valid file
```gherkin
Given I am on the scenario selector page
And I have a valid CSV file with required columns (name, sector, client_count)
When I select the file and click the upload button
Then the CSV is uploaded to the backend via POST /api/scenarios/upload-csv
And I see a success message indicating the scenario was created
And the new scenario appears in the scenario list
```

### Scenario: Invalid CSV missing required column
```gherkin
Given I am on the scenario selector page
And I have a CSV file missing the required column "name"
When I attempt to upload the file
Then I see an error message indicating the missing column "name"
And the upload is rejected (no scenario created)
```

## Examples (Specification by Example)

| CSV content (first row)                  | Action                | Expected outcome                     |
|------------------------------------------|-----------------------|--------------------------------------|
| name,sector,client_count                 | Upload valid CSV      | Scenario created successfully        |
| name,client_count (missing sector)       | Upload invalid CSV    | Error: missing column "sector"       |
| name,sector,client_count (extra column) | Upload valid CSV      | Scenario created (extra column ignored) |

## Notes
- The backend endpoint `POST /api/scenarios/upload-csv` is expected to return 201 on success with created scenario ID, and 422 with validation errors on invalid CSV.
- The UI should integrate with the existing scenario selector page, adding a file picker input and upload button.
- Upon successful upload, the scenario list should refresh (or update optimistically) to show the new scenario.
- Error handling should display validation details returned by the backend (e.g., missing columns, invalid data types).
