# S2.4 Scope — CSV Upload Frontend

## In Scope
- File picker UI in the scenario selector page
- Integration with the backend `POST /api/scenarios/upload-csv` endpoint
- Display of upload success and error messages
- Basic CSV validation (displaying backend validation errors)
- Updating the scenario list after a successful upload

## Out of Scope
- Advanced CSV parsing (e.g., streaming large files) – handled by backend
- Column mapping UI for transforming columns beyond required fields
- Persistent upload history or audit trail
- Drag‑and‑drop file upload (optional enhancement)
- Authentication/authorization specifics (assume existing auth context)

## Done when
- File picker component is implemented and integrated into the scenario selector page
- Successful upload triggers the backend endpoint and shows a success message
- Error responses from the backend (validation errors) are displayed in the UI
- Unit tests for the upload component exist and pass
- Manual QA verifies the flow with both valid and invalid CSV files
- Code adheres to project linting and formatting standards
