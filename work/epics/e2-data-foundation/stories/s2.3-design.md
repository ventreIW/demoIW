# s2.3 Design — Scenario Selector UI

---

## Component tree

```
/scenarios
  ScenarioSelectorPage (server component — fetches initial list)
    ScenarioGrid (client component — handles activation state)
      ScenarioCard × N
        SectorBadge
        StatusBadge
        Button "Seleccionar" / "Activo"
      Button "Generar nuevo" (disabled, href="#")
      Button "Subir CSV" (disabled, href="#")
```

Fetch strategy: initial list via server component (`fetch` with `cache: 'no-store'`). Activation via client-side `fetch` + `router.refresh()` to re-run server component.

## Route guard (middleware)

```typescript
// src/middleware.ts
export function middleware(request: NextRequest) {
  const { pathname } = request.nextUrl
  const isProtected = pathname.startsWith("/operations") || pathname.startsWith("/executive")
  const activeScenario = request.cookies.get("active_scenario_id")
  if (isProtected && !activeScenario) {
    return NextResponse.redirect(new URL("/scenarios", request.url))
  }
}
```

The active scenario ID is stored in a cookie by the API client after a successful activate call.
