## Backend Contract (assumed)

`POST /api/v1/scenarios/upload-csv`
- **Request:** `multipart/form-data` with field `file` (CSV file)
- **Success (201):** `ScenarioSummary` JSON — `{ id, name, sector, status: "inactive", client_count: 0, created_at }`
- **Validation error (422):** `{ "detail": [{ "loc": [...], "msg": "...", "type": "value_error" }] }`
- **CSV format:** Header row `name,sector,client_count` + one data row
- **Valid sector values:** `manufacturing`, `retail`, `professional_services`

The frontend does NOT validate CSV contents — it sends the file and displays whatever the backend returns.

**`uploadCsv()`** in `lib/api/scenarios.ts`:

```typescript
export async function uploadCsv(file: File): Promise<ScenarioSummary> {
  const formData = new FormData()
  formData.append("file", file)
  const res = await fetch(`${API_BASE}/api/v1/scenarios/upload-csv`, {
    method: "POST",
    body: formData,
  })
  if (!res.ok) {
    const body = await res.json()
    const messages = body.detail?.map((d: { msg: string }) => d.msg).join(", ")
    throw new Error(messages || `Error al subir: ${res.status}`)
  }
  return res.json()
}
```

### TC-2: Page integration — replace disabled "Subir CSV" button

**Size:** XS | **Depends on:** TC-1
**AC ref:** `s2.4-story.md` — Successful upload shows scenario in list

#### Implementation

File: `frontend/src/app/scenarios/page.tsx` (modify, lines 45-50)

Replace the disabled button block:

```tsx
// Before (lines 45-50):
<button disabled className="inline-flex h-8 ...">Subir CSV</button>

// After:
<CsvUploadWrapper />
```

Since `page.tsx` is a server component and `CsvUpload` is a client component with `useState` + file input, the integration must handle the server/client boundary:

**Option A (preferred):** Import `CsvUpload` directly — it's a `"use client"` component so Next.js handles the boundary automatically. Pass `onUploadComplete` as a server action or callback.

**Option B:** Use `dynamic(() => import("@/components/scenarios/CsvUpload"), { ssr: false })` if hydration issues arise with file inputs.

Since `page.tsx` is a server component and cannot use `useRouter`, the `onUploadComplete` callback should trigger a refresh. Two approaches:

1. **Extract to a client wrapper** — Create a small `CsvUploadWrapper` client component that imports `useRouter` and passes `router.refresh` as `onUploadComplete`:

```tsx
// In page.tsx:
import CsvUploadWrapper from "./CsvUploadWrapper"

// Replace the button block with:
<CsvUploadWrapper />

// New file: app/scenarios/CsvUploadWrapper.tsx
"use client"
import { useRouter } from "next/navigation"
import CsvUpload from "@/components/scenarios/CsvUpload"

export default function CsvUploadWrapper() {
  const router = useRouter()
  return <CsvUpload onUploadComplete={() => router.refresh()} />
}
```

2. **Server action** — less ideal for this case since we need `router.refresh()` which is client-only.

Go with Option 1 (client wrapper). One new small file — clean separation.

Files:
- `frontend/src/app/scenarios/page.tsx` — modify lines 45-50
- `frontend/src/app/scenarios/CsvUploadWrapper.tsx` — create

## Risks & Mitigations

| Risk | Likelihood | Mitigation |
|------|-----------|------------|
| Backend contract changes (Nano ships different error shape) | Low | Contract agreed upfront. Frontend parses `detail[].msg` — standard FastAPI 422 shape. If it changes, only `uploadCsv()` needs updating. |
| Server component / file input hydration | Medium | Use `"use client"` directive + client wrapper pattern (CsvUploadWrapper). Verified in s2.3 where ScenarioGrid is client-side inside server page. |
| File input styling differs across browsers | Low | Hidden input triggered by styled button — consistent across Chrome/Firefox/Safari. |
| Backend not available during development | High | Vitest mocks `uploadCsv` — all tests pass without backend. For manual testing, coordinate with Nano when endpoint is ready. |
