# s2.2 Scope — Scenario Management API

## In scope

- `app/ports/repositories.py` — `IScenarioRepository` abstract class
- `app/adapters/persistence/sqlalchemy_scenario_repo.py` — concrete implementation
- `app/application/use_cases/manage_scenario.py` — `ListScenarios`, `CreateScenario`, `ActivateScenario`, `GetActiveScenario` use cases
- `app/routers/scenarios.py` — FastAPI router, Pydantic request/response schemas
- `app/container.py` — wire `IScenarioRepository` → `SQLAlchemyScenarioRepository` via `Depends`
- `tests/test_scenarios_api.py` — integration tests for all 5 endpoints

## Out of scope

- CSV upload endpoint (s2.4)
- `IClientRepository` or any other repository (s2.1 has models only)
- Scenario data generation (E3)
- Frontend (s2.3)
