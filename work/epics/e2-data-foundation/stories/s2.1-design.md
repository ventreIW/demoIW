# s2.1 Design — Domain Schema & Migrations

**Updated:** 2026-06-25 (gemba walk against E1 shipped code)
**Complexity:** Moderate

---

## Problem & Value

**Problem:** The backend has no domain entities, no ORM models, and no database tables. Every subsequent story (API, UI, scoring, communications) requires this foundation to exist.

**Value:** After this story `alembic upgrade head` creates all 7 tables with correct constraints, 3 seeded blank scenarios are available, and the domain/persistence boundary is established so future stories only need to implement repositories.

---

## Gemba Findings (E1 code state)

| Finding | Impact on Design |
|---|---|
| `env.py` imports `Base` from `app.infrastructure.database` — but `Base` is not defined there | **Must add** `Base = DeclarativeBase()` to `database.py` as first step |
| `psycopg2-binary` not in `requirements.txt` | `alembic upgrade head` fails without it; env.py converts asyncpg URL → psycopg2 URL for sync migrations |

| `app/domain/` and `app/adapters/` don't exist | Create from scratch — net-new |
| Alembic chain: `0001` has `down_revision = None` | `0002` → `down_revision = "0001"`, `0003` → `down_revision = "0002"` |

**Legacy sweep:** Net-new. No V1 is orphaned. `app/infrastructure/database.py` gets one addition (`Base`); nothing is replaced.

---

## Approach

Add `Base` to `database.py` (where env.py expects it), then build domain entities, ORM models, mappers, and migrations as layered additions. Fix missing deps at the start.

**Lean gates:**
- KISS: No repository interfaces in this story (s2.2 scope). ORM models are plain mapped classes, no custom type adapters beyond enum-as-string.
- DRY: Enums defined once in `app/domain/enums.py`, imported everywhere.
- YAGNI: No `total_outstanding` computed column — computed at query time in s2.2.
- MVP: No repository, no API. Only schema + seed + typed domain layer.

**Drift risks:**
- AG4: Touches `database.py` + 3 new directories + alembic — 4 modules. All loosely coupled; no shared state risk.
- AG2: None — all new code.

```python
# app/adapters/persistence/models.py
from uuid import uuid4
from datetime import datetime
from sqlalchemy import ForeignKey, String, Integer, Float, JSON
from sqlalchemy.dialects.postgresql import UUID as PG_UUID, JSONB
from sqlalchemy.orm import DeclarativeBase, Mapped, mapped_column
from app.infrastructure.database import Base

class ScenarioORM(Base):
    __tablename__ = "scenarios"
    id: Mapped[str] = mapped_column(PG_UUID(as_uuid=False), primary_key=True, default=lambda: str(uuid4()))
    name: Mapped[str] = mapped_column(String(200))
    sector: Mapped[str] = mapped_column(String(50))
    seed: Mapped[int | None] = mapped_column(Integer, nullable=True)
    parameters: Mapped[dict] = mapped_column(JSONB, default=dict)
    source: Mapped[str] = mapped_column(String(50))
    status: Mapped[str] = mapped_column(String(20), default="inactive")
    created_at: Mapped[datetime] = mapped_column(server_default="now()")

class ClientORM(Base):
    __tablename__ = "clients"
    id: Mapped[str] = mapped_column(PG_UUID(as_uuid=False), primary_key=True, default=lambda: str(uuid4()))
    scenario_id: Mapped[str] = mapped_column(ForeignKey("scenarios.id", ondelete="CASCADE"))
    name: Mapped[str] = mapped_column(String(200))
    sector_description: Mapped[str | None] = mapped_column(String(500), nullable=True)
    payment_history_pattern: Mapped[str] = mapped_column(String(50))
    # total_outstanding computed via invoice join — not stored

# InvoiceORM, PaymentORM, ScoreORM, CommunicationORM, ContactResultORM follow same pattern
```

## Mapper Pattern

```python
# app/adapters/persistence/mappers.py
from uuid import UUID
from app.adapters.persistence.models import ScenarioORM
from app.domain.entities.scenario import Scenario
from app.domain.enums import Sector, ScenarioStatus

def scenario_orm_to_domain(orm: ScenarioORM) -> Scenario:
    return Scenario(
        id=UUID(orm.id),
        name=orm.name,
        sector=Sector(orm.sector),
        seed=orm.seed,
        parameters=orm.parameters,
        source=orm.source,
        status=ScenarioStatus(orm.status),
        created_at=orm.created_at,
    )

def scenario_domain_to_orm(domain: Scenario) -> ScenarioORM:
    return ScenarioORM(
        id=str(domain.id),
        name=domain.name,
        sector=domain.sector.value,
        seed=domain.seed,
        parameters=domain.parameters,
        source=domain.source,
        status=domain.status.value,
        created_at=domain.created_at,
    )
```

## Migration Structure

```python
# alembic/versions/0002_domain_schema.py
revision = "0002"
down_revision = "0001"

def upgrade() -> None:
    op.create_table("scenarios",
        sa.Column("id", postgresql.UUID(as_uuid=False), primary_key=True),
        sa.Column("name", sa.String(200), nullable=False),
        sa.Column("sector", sa.String(50), nullable=False),
        sa.Column("seed", sa.Integer, nullable=True),
        sa.Column("parameters", postgresql.JSONB, nullable=False, server_default="{}"),
        sa.Column("source", sa.String(50), nullable=False),
        sa.Column("status", sa.String(20), nullable=False, server_default="inactive"),
        sa.Column("created_at", sa.TIMESTAMP(timezone=True), server_default=sa.text("now()")),
    )
    # ... clients, invoices, payments, scores, communications, contact_results
    op.execute("""
        CREATE UNIQUE INDEX uix_scenarios_active
        ON scenarios (status)
        WHERE status = 'active'
    """)

def downgrade() -> None:
    op.execute("DROP INDEX IF EXISTS uix_scenarios_active")
    op.drop_table("contact_results")
    # ... drop in reverse FK order
    op.drop_table("scenarios")
```

## Test Fixture Update (conftest.py)

Add a `db_session` fixture for ORM unit tests (SQLite in-memory with table creation):

```python
# tests/conftest.py — ADD alongside existing client fixture
from app.infrastructure.database import Base

@pytest.fixture
async def db_session() -> AsyncGenerator[AsyncSession, None]:
    """SQLite in-memory session with all tables created. For ORM/mapper unit tests."""
    engine = create_async_engine("sqlite+aiosqlite://", echo=False)
    async with engine.begin() as conn:
        await conn.run_sync(Base.metadata.create_all)
    async with async_sessionmaker(engine, class_=AsyncSession, expire_on_commit=False)() as session:
        yield session
    await engine.dispose()
```

> Integration migration tests (alembic upgrade/downgrade) require real PostgreSQL and run locally only — not in CI.

## Acceptance Criteria

**MUST:**
1. `alembic upgrade head` (against real PostgreSQL) creates all 7 tables with FKs and the partial unique index on `scenarios(status) WHERE status='active'`
2. After upgrade, `SELECT count(*) FROM scenarios` returns 3 (seed scenarios)
3. `alembic downgrade -1` (from 0003) removes seed rows; `downgrade -1` again (from 0002) drops all 7 tables
4. `pytest tests/domain/ tests/adapters/` passes with SQLite in-memory — no PostgreSQL required
5. `mypy app/` exits 0 (all new modules type-annotated, strict)
6. ADR-002 written in `dev/decisions/`

**SHOULD:**
7. `ruff check app/` exits 0 on new modules
8. Round-trip mapper test: `scenario_orm_to_domain(scenario_domain_to_orm(s)) == s` for all entity types

**MUST NOT:**
- Import ORM models anywhere in `app/domain/` (domain layer stays ORM-free)
- Use `relationship()` in ORM models — repositories handle joins explicitly (simpler, avoids lazy-load surprises)
- Define `Base` anywhere other than `app/infrastructure/database.py`
