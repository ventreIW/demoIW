**Epic:** E2 — Data Foundation

**Owner:** —  
**Depends on:** s2.1

---

## User story

> As a demo facilitator, I want to list, create, and activate scenarios via the API so that the frontend can present the correct scenario state and the application always knows which dataset is active.

---

## Acceptance criteria

- [ ] `GET /api/scenarios` returns list of all scenarios with: `id`, `name`, `sector`, `status`, `client_count`, `created_at`
- [ ] `POST /api/scenarios` creates a blank scenario and returns its summary (status 201)
- [ ] `GET /api/scenarios/{id}` returns full scenario detail; returns 404 for unknown ID
- [ ] `PATCH /api/scenarios/{id}/activate` sets scenario to active, deactivates any currently active scenario
- [ ] `GET /api/scenarios/active` returns the active scenario or 404 if none
- [ ] Activating an already-active scenario is idempotent (returns 200, no error)
- [ ] All endpoints have integration tests covering happy path + error cases
- [ ] `ScenarioRepository` (SQLAlchemy implementation of `IScenarioRepository`) is used by all endpoints via DI
- [ ] OpenAPI schema is valid (`ruff check` passes and FastAPI generates schema without errors)
