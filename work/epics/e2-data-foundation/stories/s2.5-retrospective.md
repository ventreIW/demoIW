# Retrospective: S2.5 — CSV Upload Backend

**Dates:** 2026-07-01 → 2026-07-01
**Estimated:** S (3 SP) | **Actual:** ~1 session

## Summary
Backend endpoint `POST /api/v1/scenarios/upload-csv` que faltó en s2.4. Acepta CSV multipart/form-data con columnas `client_name`, `amount`, `due_date`, `invoice_id`. Valida columnas, parsea con `csv.DictReader`, y crea Scenario + Client + Invoice en una sola transacción vía `IScenarioRepository.create_from_csv()`. 4 integration tests cubren happy path, missing columns, empty file, y malformed CSV.

## What went well
- TDD RED-GREEN-REFACTOR: 4 tests escritos primero (RED), implementación en un solo paso (GREEN), sin necesidad de refactor mayor
- Repository pattern: `create_from_csv` encapsula la creación multi-entidad en una transacción, manteniendo el router limpio
- Contrato frontend respetado: el endpoint usa exactamente el mismo formato que el frontend de s2.4 espera (`detail: [{msg: string}]` en 422, `ScenarioSummary` en 201)
- 39/39 tests pasan, mypy limpio (28 files), sin regresiones

## What to improve
- El smoke test manual (T2) quedó bloqueado por falta de PostgreSQL en WSL. Los integration tests con SQLite verificaron el endpoint, pero una prueba real contra PostgreSQL habría detectado posibles diferencias de dialecto
- Stale `__pycache__` causó `TypeError: Router.__init__() got an unexpected keyword argument 'on_startup'` — error confuso que tomó tiempo diagnosticar. Solución: limpiar `__pycache__` de los directorios modificados
- `rai gate check gate-tests --scope` no funcionó para la estructura `backend/tests/` — tuvo que caer a `pytest` directo

## Heutagogical Checkpoint
1. **What did I learn?** FastAPI `UploadFile` + `multipart/form-data` requiere `python-multipart`. El patrón de repositorio escala bien para operaciones multi-entidad: un solo método `create_from_csv` mantiene la atomicidad de la transacción. Stale `__pycache__` es una fuente silenciosa de errores confusos después de modificar código — limpiar los directorios `app/` afectados es el fix.
2. **What would I change about the process?** Para stories donde el backend requiere PostgreSQL y no está disponible, documentar el bloqueo desde el plan en vez de descubrirlo en T2. Los integration tests con SQLite fueron suficientes para este caso.
3. **Framework improvements?** `rai gate check` necesita soportar mejor estructuras de proyecto con subdirectorios (backend/, frontend/). El `--scope` falló con `backend/tests/`.
4. **More capable of now?** Endpoints FastAPI con file upload, parsing de CSV con stdlib, creación multi-entidad transaccional en repositorio.

## Improvements applied
- `python-multipart==0.0.32` agregado a `requirements.txt`
- `__pycache__` limpiado de `backend/app/routers/` para resolver error de importación

## Patterns added / reinforced
- **Nuevo patrón:** Después de modificar código en un subdirectorio `app/`, limpiar `__pycache__` de ese subdirectorio (`find backend/app -name "__pycache__" -exec rm -rf {} +`). El bytecode compilado puede retener imports y firmas de funciones obsoletas que producen errores confusos.
- **Reforzado:** Repository pattern con método de operación compuesta (`create_from_csv`) — seguir encapsulation sobre exposición de implementación
