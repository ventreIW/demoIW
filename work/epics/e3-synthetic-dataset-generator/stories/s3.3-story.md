# s3.3 — LLM Enrichment Layer

**Epic:** E3 — Synthetic Dataset Generator  
**Branch:** `s3.3-llm-enrichment` (from `e3-synthetic-dataset-generator`)  
**Size:** S  
**Owner:** —  
**Depends on:** s3.1 (RawDataset) + s3.2 (ILLMPort)

---

## User story

> As a demo facilitator, I want generated companies to have sector-appropriate names and descriptions instead of generic Faker names so that the demo feels realistic and convincing to prospects.

---

## Acceptance criteria

- [ ] `LLMEnrichmentService.enrich(raw_dataset, model)` replaces `name` and adds `sector_description` on all client rows
- [ ] Enrichment is batched: one LLM call per 20 clients (configurable)
- [ ] If LLM call fails, enrichment falls back to the original Faker names (graceful degradation — demo must not break)
- [ ] Prompt template is loaded from `prompts/data_enrichment/v1_company_description.txt` — not hardcoded in the service (RF-04.6 pattern)
- [ ] Generated company names in the output are NOT the same as the original Faker names (enrichment actually ran)
- [ ] Unit tests use a mock `ILLMPort` — no real OpenRouter calls
- [ ] `GenerateDataset` use case wires together `ProceduralGenerator` + `LLMEnrichmentService`
- [ ] ADR-004 written (batching strategy)
