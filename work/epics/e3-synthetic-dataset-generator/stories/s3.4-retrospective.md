## Heutagogical Checkpoint

1. **What did I learn?** shadcn/ui Select components only render option text in the DOM after the dropdown is opened via click. This means test assertions for default option text fail unless the dropdown is triggered first. Also, form inputs with `type="number"` and `id` attributes are more reliably found via `getByLabelText` than `getByRole('spinbutton')`.

2. **What would I change about the process?** For stories with external backend dependencies (DB, API keys), the manual integration test should be marked as a post-condition item rather than a separate task. The frontend can be fully verified with mocked tests; the E2E test is a deployment concern.

3. **Framework improvements?** The `rai gate check` gate didn't work for frontend (Vitest) tests due to the nested project structure. The fallback procedure in the pitfalls section worked well (direct `npx vitest run`). The `rai gate check` manifest should be configured for this project.

4. **What am I more capable of now?** Confident with rewriting Next.js App Router forms with inline results display, vitest mocking patterns for `fetch`/`next/navigation`/`sonner`, and handling shadcn/ui Select components in tests.
