## User story

> As a developer, I want a seed-based procedural generator that produces statistically realistic client, invoice, and payment data so that every generated scenario is reproducible and distribution-appropriate for its sector.

---

## Acceptance criteria

- [ ] `ProceduralGenerator(params).generate()` returns a `RawDataset` with `clients`, `invoices`, `payments` DataFrames
- [ ] Calling with the same seed and params twice returns identical DataFrames (determinism test)
- [ ] `client_count` rows in the clients DataFrame match the `params.client_count` input exactly
- [ ] Invoice amounts follow a Normal distribution (mean ± std from params); no negative amounts
- [ ] Days overdue follow an Exponential distribution (lambda from params)
- [ ] `overdue_rate` fraction of invoices have `status = OVERDUE`
- [ ] Payment rows exist for all non-overdue invoices
- [ ] `payment_history_pattern` distribution differs between MANUFACTURING and RETAIL sectors (sector-specific weights)
- [ ] Unit tests cover: reproducibility, client count, distribution sanity (mean within 15% tolerance), sector weighting
- [ ] `mypy` passes on all new files
- [ ] ADR-003 written
