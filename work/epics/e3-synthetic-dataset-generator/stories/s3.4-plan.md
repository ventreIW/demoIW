## Task List

### Task 1: Add GenerationParams type and generateScenario API function

**Description:** Add the `GenerationParams` interface to the shared types module and implement the `generateScenario` API function that sends a POST request to `/api/v1/scenarios/generate`.

**Files to modify/create:**
- `frontend/src/types/scenario.ts` — add `GenerationParams` interface
- `frontend/src/lib/api/scenarios.ts` — add `generateScenario` function
- `frontend/src/lib/api/__tests__/scenarios.test.ts` — unit tests for the API function

**TDD cycle:**
1. RED: Write test that calls `generateScenario` with mock params and asserts correct request shape + response handling
2. GREEN: Add `GenerationParams` type and `generateScenario` function
3. REFACTOR: Align naming with existing patterns

**Verification:**
```bash
rai gate check gate-tests --scope frontend/src/lib/api/__tests__/
rai gate check gate-lint
rai gate check gate-types
```

**Size:** XS
**Dependencies:** None

### Task 2: Rewrite GenerateScenarioForm to display results inline

**Description:** Replace the current auto-activate + redirect behavior with an inline results display. On successful generation, show scenario details (ID, name, sector, client count, status, timestamp), a download link, a copy-ID button, and append `?scenarioId=<id>` to the URL. The "Activate and Use" button becomes optional (not automatic). Remove the `overdue_rate` slider and add `amount_mean`/`amount_std` fields (hidden, derived from old slider value).

**Files to modify/create:**
- `frontend/src/components/scenarios/GenerateScenarioForm.tsx` — rewrite submit handler + add results display section
- `frontend/src/components/scenarios/__tests__/GenerateScenarioForm.test.tsx` — new test file

**TDD cycle:**
1. RED: Write tests for success state (scenario details visible, download link, copy ID, URL param), loading state (button disabled, spinner), error state (toast message), and "Activate and Use" button
2. GREEN: Rewrite the form component
3. REFACTOR: Clean up unused state, ensure `useRouter` mock aligns

**Verification:**
```bash
rai gate check gate-tests --scope frontend/src/components/scenarios/__tests__/
rai gate check gate-lint
rai gate check gate-types
```

**Size:** S
**Dependencies:** Task 1 (needs `generateScenario` function and `GenerationParams` type)

### Task 3: Manual integration test

**Description:** Start the backend API server, start the frontend dev server, navigate to the generate page, fill in parameters, submit, and verify:
- Scenario is generated successfully
- Results display inline with correct details
- Download link is present
- Copy-ID button works
- "Activate and Use" navigates to /operations
- Error state shows readable message

**Verification:** Manual walkthrough of the happy path, download link, activation flow, and error state.

**Size:** XS
**Dependencies:** Task 2

## Execution Order

```
Task 1 -> Task 2 -> Task 3
(API+types)  (form)  (manual test)
```

**Rationale:** Risk-first ordering. Task 1 is the foundation — without the API function and types, the form cannot compile or function. Task 2 is the core deliverable (the most complex task). Task 3 is a lightweight end-to-end validation.

## Gate Policy

| When | Command | Scope |
|------|---------|-------|
| Per task | `rai gate check gate-tests --scope <test_dir>/` | Only tests for the changed module |
| Per task | `rai gate check gate-lint` | Full project (fast) |
| Per task | `rai gate check gate-format` | Full project (fast) |
| Per task | `rai gate check gate-types` | Full project |
| End of story | `rai gate check gate-tests --scope frontend/` | Frontend package only — NOT full suite |
| MR creation | `/rai-mr-create` | Full suite — the ONLY place where unscoped tests run |

**Note:** If `rai gate check` fails or is not found after one attempt, use `pytest` directly as a fallback (`cd backend && python -m pytest <path> -v`) and continue. For frontend tests, fallback is `cd frontend && npx vitest run <path>`.

## Risks and Mitigations

| Risk | Likelihood | Mitigation |
|------|-----------|------------|
| Backend `/api/v1/scenarios/generate` not yet implemented | Medium | Use mocked API responses in tests; manual test blocked until backend is available |
| Download endpoint `/api/v1/scenarios/{id}/download` not available | Low | Show download link anyway; it will 404 until backend implements it |
| `activateScenario` API changes | Low | Import and call existing function; interface is already consumed in `ScenarioCard` |
