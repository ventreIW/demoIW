## User story

> As a developer, I want a robust OpenRouter adapter that implements the ILLMPort interface so that all LLM-dependent features (enrichment, communications, NL query) can share a single, configuration-driven AI gateway without coupling to any specific model.

---

## Acceptance criteria

- [ ] `OpenRouterAdapter` implements `ILLMPort` with `generate()` and `query()` methods
- [ ] Model name is always read from the caller (never hardcoded inside the adapter) (GR-AI-002)
- [ ] A transient 5xx from OpenRouter is retried up to 3 times with exponential backoff
- [ ] A 4xx error (bad request, auth failure) raises `ExternalServiceError` immediately without retry
- [ ] Every LLM call produces a structured log entry with: `model`, `prompt_tokens`, `completion_tokens`, `latency_ms`
- [ ] Unit tests use `respx` (httpx mock library) — no real network calls in CI
- [ ] `OPENROUTER_API_KEY` and `OPENROUTER_BASE_URL` are read from `Settings`; added to `.env.example`
- [ ] `mypy` passes (typed request/response models)
