# s3.4 Scope — Generate Scenario Endpoint & UI (backend)

## In Scope

- `app/ports/repositories.py` — `IClientRepository` + `IInvoiceRepository` + `IPaymentRepository` interfaces
- `app/adapters/persistence/sqlalchemy_client_repo.py`, `sqlalchemy_invoice_repo.py`, `sqlalchemy_payment_repo.py`
- `GenerateDataset` use case updated to persist via repositories (not just return RawDataset)
- `POST /api/scenarios/generate` router endpoint
- `app/container.py` — wire new repositories + ports
- `tests/integration/test_generate_scenario.py`

## Out of Scope

- Scoring / ML training on the generated data (E4)
- Communications generation (E5)
- CSV upload (already done in s2.4)
- Streaming progress (WebSocket) — spinner only
- Frontend form, loading state, UI components (handled in separate frontend story)

## Done when

- `POST /api/scenarios/generate` returns 201 + ScenarioSummary for valid input
- Generated scenario is persisted: clients, invoices, payments appear in the database
- Same seed produces identical client names and invoice amounts (determinism)
- Endpoint responds within 30 seconds for 100 clients (NFR-02)
- Integration test passes: `POST /api/scenarios/generate` → asserts scenario + 3 client rows in DB
- All unit and integration tests for the backend components pass
- MyPy and type checking pass for backend changes
