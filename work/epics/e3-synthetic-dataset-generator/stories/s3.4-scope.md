# s3.4 Scope — Generate Scenario Frontend

## In Scope
- React component for the scenario generation form
- State management for form inputs and submission
- API call to the backend endpoint `POST /api/scenarios/generate`
- Display of loading state during request
- Display of success message and download link or JSON output on success
- Display of error messages for validation and API errors
- Unit tests for the component (using Jest and React Testing Library)
- Integration tests (if applicable) for the form submission flow

## Out of Scope
- Backend implementation of the `/api/scenarios/generate` endpoint (handled in backend story)
- Styling beyond the basic layout (assumes existing design system)
- Data persistence on the frontend (beyond the session)
- Advanced features like saving scenarios to user account
- Export to formats other than JSON (e.g., CSV, PDF)

## Done when
- The form is rendered correctly with all required fields
- Submitting the form with valid data calls the backend API and displays the result
- Submitting the form with invalid data shows appropriate validation errors
- Loading state is shown during the API request
- Unit tests pass for the component
- Manual QA confirms the workflow works as expected
