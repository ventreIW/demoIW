# s3.1 Scope — Procedural Data Generation Layer

## In scope

- `app/application/use_cases/generate_dataset.py` — `GenerationParams` model
- `app/adapters/dataset/procedural_generator.py` — `ProceduralGenerator` (implements `IDatasetPort`)
- `app/ports/dataset_port.py` — `IDatasetPort`, `RawDataset` types
- `app/domain/value_objects/raw_dataset.py` — `RawDataset(clients, invoices, payments)`
- Sector-specific distribution config (hardcoded constants in generator, not DB-driven)
- `tests/unit/test_procedural_generator.py` — all acceptance criteria tests
- ADR-003 in `dev/decisions/`

## Out of scope

- LLM enrichment (s3.3)
- OpenRouter adapter (s3.2)
- Any API endpoint
- Persistence to database (s3.4)
- Frontend
