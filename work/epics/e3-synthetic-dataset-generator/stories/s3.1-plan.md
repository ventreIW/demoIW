### T2 — ProceduralGenerator implementation (M) — riskiest

The statistical core: determinism, distributions, sector weighting. Same test file grows.

- **Files (create):** `app/adapters/dataset/procedural_generator.py` — `ProceduralGenerator(IDatasetPort)`
  with `_generate_clients`, `_generate_invoices`, `_generate_payments`, `_sector_payment_weights`.
- **TDD (RED → GREEN), one test per AC scenario:**
  1. `test_determinism` — same seed+params twice → `pd.testing.assert_frame_equal` on all 3 frames.
  2. `test_client_count_matches_params` — `clients.shape[0] == params.client_count`.
  3. `test_amounts_positive` — `(invoices.amount > 0).all()`.
  4. `test_amount_mean_within_tolerance` — sample mean within 15% of `amount_mean` (large N). *(SHOULD)*
  5. `test_overdue_rate_within_tolerance` — overdue fraction within 5% of `overdue_rate` (large N).
  6. `test_payment_row_per_settled_invoice` — every non-overdue invoice id ∈ payments.invoice_id.
  7. `test_sector_weighting` — MANUFACTURING DELAYED_30 share > RETAIL (large N).
- **Determinism anchors:** `np.random.default_rng(seed)` + `Faker.seed(seed)`.
- **Sector weights:** hardcoded table from design (rows sum to 1.0, real `PaymentPattern` values).
- **Verification:** `rai gate check gate-tests --scope backend/tests/adapters/`, `gate-lint`, `gate-format`, `gate-types`.

## Execution order

```
T0 (deps) → T1 (scaffolding) → T2 (generator + stats) → T3 (ADR + smoke)
```

Linear — each task's tests need the prior task's types/deps. Acyclic. Risk-first within the
constraint: T2 (statistical correctness + determinism) is the riskiest and is tackled as soon
as its scaffolding (T0/T1) exists.

## Risks & mitigations

| Risk | Mitigation |
|------|------------|
| Non-determinism from unseeded Faker | Seed both `default_rng` and `Faker.seed` (D5); `test_determinism` guards it |
| Flaky distribution assertions on small N | Use large N (e.g. ≥2000 invoices) + generous tolerance (5%/15%) in stats tests |
| Sector-weight rows not summing to 1.0 | `np.random.Generator.choice` requires `p` sum≈1; assert in a guard test |
| `numpy`/`pandas` version drift vs Python 3.12 | Pin compatible versions in T0; verify import before impl |
| RawDataset column drift vs s3.4 needs | Columns aligned to `Invoice` fields now; open question flagged in design |
