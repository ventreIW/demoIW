# Story: Generate Scenario Frontend

## User Story
As a demo facilitator,
I want to see a form where I can select a sector and input parameters (client count, seed, etc.),
click a "Generate Scenario" button,
and see the generated scenario displayed (or a link to download it),
so that I can quickly demonstrate the product to prospects.

## Acceptance Criteria

### Scenario: Successful generation with valid inputs
```gherkin
Given the user is on the scenario generation page
When they select a sector, enter a client count of 100, and a seed of 42
And they click the "Generate Scenario" button
Then they should see a success message and a link to download the generated scenario
And the URL should include the generated scenario ID
```

### Scenario: Error handling for missing required field
```gherkin
Given the user is on the scenario generation page
When they leave the client count field empty
And they click the "Generate Scenario" button
Then they should see an error message indicating the client count is required
```

## Examples (Specification by Example)
| Input | Action | Expected Output |
|-------|--------|-----------------|
| Sector: Healthcare, Client Count: 50, Seed: 123 | Submit the form | Success: scenario generated and shown |
| Sector: Finance, Client Count: 0, Seed: 999 | Submit the form | Error: client count must be positive |
| Sector: , Client Count: 100, Seed: 42 | Submit the form | Error: sector is required |
## Notes
- This story focuses on the frontend implementation (React/TypeScript components, state management, API calls).
- The backend endpoint `POST /api/scenarios/generate` is assumed to be available (from backend story s3.4-backend or similar).
- The form should handle loading states and errors gracefully.
- The generated scenario should be displayed in a readable format (JSON) or as a downloadable file.
- Dependencies: backend story s3.4 (or whatever the backend story ID is) must be completed first.
