# s3.4 — Generate Scenario Endpoint & UI

**Epic:** E3 — Synthetic Dataset Generator  
**Branch:** `s3.4-generate-scenario-endpoint` (from `e3-synthetic-dataset-generator`)  
**Size:** S  
**Owner:** —  
**Depends on:** s3.3

---

## User story

> As a demo facilitator, I want to fill a form with sector and parameters, click "Generar escenario," and have a fully populated scenario ready for demonstration within 30 seconds so that I can run the demo for any prospect without preparation.

---

## Acceptance criteria

- [ ] `POST /api/scenarios/generate` with valid `GenerationParams` body returns 201 + ScenarioSummary
- [ ] The generated scenario is persisted: clients, invoices, payments appear in the database
- [ ] Calling with the same seed returns the same client names and invoice amounts (determinism via API)
- [ ] Endpoint returns within 30 seconds for 100 clients (NFR-02)
- [ ] "Generar nuevo" button on scenario selector opens the generation form
- [ ] Form fields: sector (dropdown), client count (number, 10–500), invoices per client (number, 1–10), overdue rate (slider 10%–90%), seed (auto-generated, optional override)
- [ ] Submit shows loading state ("Generando datos..." spinner)
- [ ] On success: newly generated scenario becomes active; page redirects to operations panel placeholder
- [ ] On API error: toast with error message; form stays visible
- [ ] Integration test: `POST /api/scenarios/generate` → assert scenario + 3 client rows in DB
