# Story: Generate Scenario Endpoint & UI (backend)

## User Story
As a demo facilitator,
I want to fill a form with sector and parameters, click "Generar escenario," and have a fully populated scenario ready for demonstration within 30 seconds,
so that I can run the demo for any prospect without preparation.

## Acceptance Criteria

### Scenario: Successful generation with valid parameters
```gherkin
Given the system is running and s3.1 and s3.3 are complete
When I send a POST request to /api/scenarios/generate with valid GenerationParams
Then the response status is 201
And the response body contains a ScenarioSummary
And the scenario is persisted in the database with clients, invoices, and payments
```

### Scenario: Deterministic generation with fixed seed
```gherkin
Given the system is running and s3.1 and s3.3 are complete
When I send two POST requests to /api/scenarios/generate with the same GenerationParams and seed
Then both responses have status 201
And the client names and invoice amounts in the first position are identical
```

## Examples (Specification by Example)

| GenerationParams.seed | GenerationParams.client_count | Expected outcome                                  |
|-----------------------|-------------------------------|---------------------------------------------------|
| 42                    | 100                           | Same client count and total outstanding on repeat |
| 123                   | 10                            | Persisted clients, invoices, payments in DB       |

## Notes
- This story covers only the backend implementation (endpoint, repositories, use case update, container wiring, integration tests).
- Frontend implementation (form, loading state) is handled in a separate frontend story by Renata.
- Depends on completion of s3.1 (procedural generation) and s3.3 (LLM enrichment).
- Non-functional requirement: endpoint must return within 30 seconds for 100 clients (NFR-02).
- Integration test must verify scenario + 3 client rows in DB.
