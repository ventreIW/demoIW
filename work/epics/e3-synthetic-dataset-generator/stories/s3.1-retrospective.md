## Heutagogical Checkpoint

1. **What did I learn?** That seeded numpy RNG determinism is undone by any unseeded entropy source
   in the same function — `uuid.uuid4()` was the trap. Deriving identity from `rng.bytes(16)` keeps
   the whole DataFrame reproducible, which `pd.testing.assert_frame_equal` then guards.
2. **What would I change about the process?** Add an explicit "verify enum/type references against
   code" sub-step to design when a pre-authored design doc exists — the doc had drifted from the
   real `enums.py`, and only the gemba walk caught it.
3. **Framework improvements?** (a) `rai session start` should sync/verify the backend venv against
   `requirements.txt`. (b) The orphaned-test check snippet in `rai-story-implement` uses a reversed
   `git diff HEAD...$base` range that produces false positives (it flagged this story's own test);
   correct form is `git diff --name-only main...HEAD`.
4. **More capable of now?** Building deterministic statistical generators under mypy strict with the
   pandas/numpy/faker stack, and pragmatically deciding where type-strictness earns its keep
   (pandas ignored at boundary; numpy/faker strict).

## Post-review refinement (2026-07-02)

After the initial review, a direct question — *"did you assume some probability distribution?"* —
surfaced the story's most important lesson: the client `payment_history_pattern` was **decorative**.
Overdue status came from a global Bernoulli independent of the pattern, so an `ON_TIME` client
could be all-overdue. For a **catalog demo** whose whole point is to show the collectability engine
(E4) working, that leaves *no signal to score*.

Fixed by making the pattern causal (ADR-004): per-pattern `_PatternProfile` drives overdue
propensity, ageing, and payment behaviour; overall overdue rate now emerges from the sector mix
(ON_TIME 0.05 → DEFAULT 0.85, monotonic). Also removed the misnamed `days_overdue_lambda`
(a scale, not a rate) and added `reference_date` for cross-day determinism.

**New pattern captured:** *synthetic-data generators for ML demos must make labelled attributes
causal, not decorative* — else downstream ML finds no signal. This is the headline learning of s3.1.
