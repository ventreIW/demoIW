# S8.14: PWA Config — Implementation Plan

**Story ID:** S8.14  
**Epic:** E8  
**Size:** M  
**Complexity:** Simple  
**Date:** 2026-07-08  
**Status:** Planned

## Overview

Add static PWA configuration to the Next.js frontend: manifest.json, service worker,
layout metadata, and app icons. No UI — pure static assets and metadata.

## Architecture Notes

- All PWA files go under `frontend/public/` (Next.js auto-serves from public/).
- The `public/` directory doesn't exist yet — net-new creation.
- Layout metadata uses Next.js `Metadata` API (type-safe, already imported).

## Gate Policy

The RaiSE gate system (`rai gate check`) runs pytest for the Python backend.
Frontend tests use vitest (Node.js). For this story, all verification uses
`npx vitest run` in the `frontend/` directory since the gate system is not
configured for the Node.js test runner.

| When | Command | Scope |
|------|---------|-------|
| Per task | `cd frontend && npx vitest run src/__tests__/pwa/` | PWA test files only |
| Per task | `cd frontend && npm run typecheck` | Full frontend project |
| Per task | `cd frontend && npm run lint` | Full frontend project |
| End of story | `cd frontend && npx vitest run` | Full frontend suite |

### Task 1: PWA manifest and icons

**Description:** Create `manifest.json` with required PWA fields and placeholder
app icons at 192px and 512px.

**Files:**
- `frontend/public/manifest.json` (create) — PWA manifest
- `frontend/public/icons/icon-192.png` (create) — 192px placeholder icon
- `frontend/public/icons/icon-512.png` (create) — 512px placeholder icon
- `frontend/src/__tests__/pwa/manifest.test.ts` (create) — manifest validation test

**TDD Cycle:**

1. RED: Write test that:
   - Reads `public/manifest.json` via `fs.readFileSync`
   - Validates `name`, `short_name`, `description` are non-empty strings
   - Validates `display` is `"standalone"`
   - Validates `theme_color` and `background_color` are valid hex colors
   - Validates `icons` array contains entries for `192x192` and `512x512` sizes
   - Validates `start_url` is `"/"`
2. GREEN: Create `manifest.json` with values from the design spec. Create
   placeholder PNG icons (solid-color rectangles, any tool available).
3. REFACTOR: Verify JSON formatting, icon sizes are correct dimensions.

**AC reference:** S8.14-story.md scenarios "User sees installable PWA in browser"
and "App loads with valid manifest"

**Verification:**
```bash
cd frontend && npx vitest run src/__tests__/pwa/manifest.test.ts
```

**Files:**

- `frontend/src/app/__tests__/layout-metadata.test.tsx` (create) — metadata validation test

**TDD Cycle:**

1. RED: Write test that:

### Task 4: Manual integration test

**Description:** Build and run the frontend, verify Lighthouse PWA audit score >= 90,
and confirm installability in the browser.

**Steps:**
1. Build the frontend: `cd frontend && npm run build`
2. Start dev server: `cd frontend && npm run dev`
3. Open in browser, run Lighthouse PWA audit
4. Verify `manifest.json` is served at `/manifest.json`
5. Verify service worker is registered in DevTools > Application > Service Workers
6. Verify install prompt appears

**AC reference:** All three Gherkin scenarios in S8.14-story.md

**Verification:**
Manual — Lighthouse PWA score >= 90, install prompt available.

**Size:** S  
**Dependencies:** Tasks 1, 2, 3 (all must be complete)

## Risks and Mitigations

| Risk | Impact | Mitigation |
|------|--------|------------|
| Placeholder icons fail Lighthouse | Audit score < 90 | Use properly-sized PNGs (192×192, 512×512); Lighthouse checks dimensions, not design quality |
| SW registration fails in dev mode | Install prompt not shown | Next.js dev server serves static files; verify with `curl localhost:3000/sw.js` |
| Metadata not picked up by Next.js | Manifest link missing from HTML | Test metadata export directly; Next.js injects `manifest` into `<head>` automatically |
| Railway deployment path issues | Manifest/SW not served in prod | All assets are in `public/` — Next.js serves them at root; no path config needed |
