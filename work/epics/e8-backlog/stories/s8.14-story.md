# Story: PWA Config

## User Story
As a **user**,
I want demoIW to be installable as a Progressive Web App,
so that I can install it on my device and use it offline.

## Acceptance Criteria

### Scenario: User sees installable PWA in browser
```gherkin
Given the user has opened demoIW in Chrome or Edge
When they open the browser's install prompt
Then the app should appear as installable
And the install button should be enabled
```

### Scenario: App loads with valid manifest
```gherkin
Given the user visits demoIW in their browser
When the page loads
Then a valid manifest.json is present
And it contains name, icons, and standalone display mode
And the theme and background colors are defined
```

### Scenario: Service worker is registered
```gherkin
Given demoIW is loaded in the browser
When the page initializes
Then a service worker is registered and active
And it responds to fetch events
```

## Examples (Specification by Example)

| Requirement        | Expected Value                    | Verification Method    |
|--------------------|-----------------------------------|------------------------|
| manifest.json      | Present in app root               | File exists check      |
| App name           | demoIW or appropriate name        | manifest.name field    |
| Icons              | 192px and 512px icons present    | manifest.icons array   |
| Display mode       | standalone                        | manifest.display       |
| Theme color        | Defined (e.g., #4CAF50)          | manifest.theme_color   |
| Service worker     | Registered and functional           | devtools registration  |
| Lighthouse PWA     | Score >= 90                       | Lighthouse audit       |

## Notes
- No UI configuration needed - this is static PWA setup
- Focus on installability requirements only
- Service worker should be minimal (cache-busting for dev, installability for prod)
