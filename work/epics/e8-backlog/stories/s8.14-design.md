# Design: PWA Config (S8.14)

## Problem
demoIW is a web application without Progressive Web App capabilities. Users cannot install it on their devices for offline use or native-like experience.

## Value
Users can install demoIW on their devices (Chrome/Edge) and have a native app-like experience with standalone display mode.

## Approach
Add static PWA configuration to the Next.js frontend by:
1. Creating `public/manifest.json` with required PWA fields
2. Creating a basic service worker (`public/sw.js`) for installability
3. Updating `app/layout.tsx` to link the manifest and set theme colors
4. Creating app icons (192px and 512px)

No UI configuration needed — this is a one-time setup for the application.

## Components

| Name | Change | Purpose |
|------|--------|---------|
| `public/manifest.json` | create | PWA manifest with name, icons, colors, display mode |
| `public/sw.js` | create | Basic service worker for installability |
| `public/icons/icon-192.png` | create | 192px app icon |
| `public/icons/icon-512.png` | create | 512px app icon |
| `app/layout.tsx` | modify | Add manifest link, theme color, and viewport settings |

## Legacy Sweep
nothing — this is net-new PWA setup; no existing V1 components to orphan.

## Acceptance Criteria

| ID | Description | Severity |
|----|-------------|----------|
| AC-1 | manifest.json exists at app root with valid structure | must |
| AC-2 | manifest contains name, short_name, icons, theme_color, background_color, display: "standalone" | must |
| AC-3 | Service worker is registered and functional | must |
| AC-4 | Lighthouse PWA audit score >= 90 | must |
| AC-5 | App appears installable in Chrome/Edge browser | must |

## Examples

### manifest.json structure
```json
{
  "name": "demoIW",
  "short_name": "demoIW",
  "description": "Industrial wastewater treatment decision support",
  "start_url": "/",
  "display": "standalone",
  "background_color": "#ffffff",
  "theme_color": "#4CAF50",
  "icons": [
    {
      "src": "/icons/icon-192.png",
      "sizes": "192x192",
      "type": "image/png"
    },
    {
      "src": "/icons/icon-512.png",
      "sizes": "512x512",
      "type": "image/png"
    }
  ]
}
```

### Service worker (sw.js)
```javascript
self.addEventListener('install', (event) => {
  self.skipWaiting()
})

self.addEventListener('activate', (event) => {
  event.waitUntil(clients.claim())
})

self.addEventListener('fetch', (event) => {
  // Basic fetch handler for installability
  return
})
```

### Updated layout.tsx head section
```tsx
import type { Metadata } from 'next'
import '@/styles/globals.css'

export const metadata: Metadata = {
  title: 'demoIW',
  description: 'Industrial wastewater treatment decision support',
  manifest: '/manifest.json',
  themeColor: '#4CAF50',
  viewport: 'width=device-width, initial-scale=1, maximum-scale=1',
}
```

## Decisions

| ID | Title | Rationale |
|----|-------|-----------|
| D-1 | Use public folder for static assets | Next.js serves from public/ automatically, no additional config needed |
| D-2 | Minimal service worker | Just enough for installability; no caching logic needed per scope |
| D-3 | Green theme color | Matches typical demoIW branding (can adjust to project palette) |
