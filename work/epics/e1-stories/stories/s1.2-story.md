# Story S1.2: Frontend Skeleton

## User Story
As a developer
I need a basic frontend structure using Next.js 15 + App Router
So that I can build consistent UI components on top of a solid foundation

## Stack
- Next.js 15 (App Router)
- TypeScript (strict mode)
- Tailwind CSS
- shadcn/ui components

## Approach
- `app/layout.tsx` → Root layout with sidebar + main content shell
- `app/page.tsx` → Home page placeholder
- Sidebar navigation using shadcn/ui `Sidebar` component with `NavigationMenu`
- 2 sample routes: Home (`/`) and a placeholder `/about`

## Components
| Component | Type | Purpose |
|-----------|------|---------|
| `components/layout/sidebar.tsx` | create | Collapsible sidebar with nav links |
| `components/layout/header.tsx` | create | Top bar with page title/actions |
| `components/ui/button.tsx` | shadcn | Button primitives |
| `components/ui/sheet.tsx` | shadcn | Mobile sidebar overlay |
| `app/layout.tsx` | modify | Root layout with sidebar shell |
| `app/page.tsx` | modify | Home page with placeholder content |
| `app/about/page.tsx` | create | About page placeholder |

## Acceptance Criteria

**MUST:**
1. `app/layout.tsx` renders sidebar + main content shell
2. Sidebar has navigation links to `/` and `/about`
3. `app/page.tsx` renders placeholder home content
4. `app/about/page.tsx` renders placeholder about content
5. TypeScript strict passes (`tsc --strict`)
6. Tailwind classes are valid

**SHOULD:**
1. Sidebar responsive (collapsible on mobile via Sheet component)
2. Active route highlighted in sidebar

**MUST NOT:**
1. Use React Router or any client-side router (Next.js handles this)
2. Implement real page content (placeholders only)
3. Add complex state management or data fetching

## Notes
- No legacy sweep needed (greenfield)
- shadcn/ui components must be installed via CLI before use
