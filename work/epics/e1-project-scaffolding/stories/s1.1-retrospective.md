## Heutagogical Checkpoint

1. **What did I learn?**
   - YAML double-quoted multiline strings fold newlines to spaces, but single quotes within a YAML double-quoted string still cause shell quoting issues when the command string is passed to a subprocess. The diagnosis took two attempts.
   - `eslint-config-next` v9 requires flat config (`eslint.config.mjs`); the old `.eslintrc.json` format is not supported.
   - `"type": "module"` in `package.json` is required for Prettier and ESLint to parse `.js`/`.mjs` config files without warnings in ESM-only mode.
   - `pytest-asyncio >= 0.23` requires explicit `asyncio_default_fixture_loop_scope` to suppress the deprecation warning.

2. **What would I change about the process?**
   - Include "gate command configuration" as an explicit component in the s1.1 design (not just implied by "update manifest.yaml"). The design component list had `manifest.yaml` but not the scripts/ directory.
   - Add a supply-chain note for `uv` as the venv tool of choice on this system — it's already installed and `python3 -m venv` fails without `python3-venv` package.

3. **Framework improvements?**
   - The `rai-story-design` skill should prompt: "Does this story modify manifest.yaml? If so, plan for `scripts/*.sh` gate wrappers (PAT-1)."
   - The bootstrapping exception (PAT-2) is now documented — future scaffolding stories can reference it explicitly in the plan.

4. **More capable of now?**
   - Full monorepo tooling baseline setup from scratch: ruff + mypy strict + pytest-asyncio, Next.js 15 TypeScript strict + ESLint flat config + Prettier ESM, GitHub Actions parallel CI, `rai gate check` wired end-to-end.
   - Can diagnose YAML-to-shell quoting issues and apply the script-file workaround immediately.
