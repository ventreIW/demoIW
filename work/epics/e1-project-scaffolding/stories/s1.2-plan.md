## Tasks

### T1: Healthcheck route + Tailwind + Root layout (XS) — TDD

**Why first:** Healthcheck is the only testable behavior. TDD from RED to GREEN surfaces any Vitest/Next.js config issues early. Tailwind + layout are mechanical setup with no test dependencies.

**Files to create:**
- `frontend/src/app/healthcheck/route.test.ts` — RED: test expects `{ status: "ok" }`, fails because route.ts doesn't exist
- `frontend/src/app/healthcheck/route.ts` — GREEN: `NextResponse.json({ status: "ok" })`
- `frontend/src/styles/globals.css` — Tailwind directives (`@tailwind base/components/utilities`)
- `frontend/tailwind.config.ts` — content paths, extends theme
- `frontend/postcss.config.mjs` — tailwindcss + autoprefixer plugins
- `frontend/src/app/layout.tsx` — RootLayout: `lang="es"`, imports globals.css, renders `{children}`
- `frontend/next.config.ts` — Next.js config (empty/default)

**AC reference:** `s1.2-story.md` — Happy path: "healthcheck returns { status: 'ok' }"; "dev server starts without errors"

**TDD cycle:**
1. Write `route.test.ts` → RED (import fails — no route.ts yet)
2. Create `route.ts` → GREEN (test passes)
3. No refactor needed at this scale

**Verification:**
```bash
cd frontend && npm test -- --run          # 1 test passes (healthcheck)
cd frontend && npm run typecheck           # exits 0
cd frontend && npm run lint                # exits 0
cd frontend && npm run dev &              # starts on port 3000, no errors — kill after verify
```

**Size:** XS | **Dependencies:** None (s1.1 scaffolding already done)

### T2: Layout components + shadcn/ui + E2E validation (XS)

**Why second:** Depends on T1 (layout.tsx, globals.css must exist). shadcn/ui init requires the Tailwind setup from T1.

**Files to create:**
- `frontend/src/components/layout/MainLayout.tsx` — flex container: `<Sidebar>` + `<main className="flex-1 p-6">{children}</main>`
- `frontend/src/components/layout/Sidebar.tsx` — fixed sidebar (w-64, md:block hidden) with nav link placeholders (Operaciones, Ejecutivo, Escenarios) + logo placeholder
- `frontend/src/app/page.tsx` — renders `<MainLayout>` with placeholder content
- `frontend/src/lib/utils.ts` — `cn()` helper (`clsx` + `tailwind-merge`)

**shadcn/ui setup:**
```bash
cd frontend
npx shadcn@latest init  # style=default, base=slate, CSS variables=yes
npx shadcn@latest add button card separator skeleton
```

**Files modified by shadcn:**
- `frontend/src/styles/globals.css` — CSS variables added
- `frontend/components.json` — shadcn config
- `frontend/src/components/ui/` — generated components (do not edit)

**AC reference:** `s1.2-story.md` — Happy path: "layout shell with sidebar placeholder and main area"; Edge case: "404 page on undefined route"

**Verification:**
```bash
cd frontend && npm run dev &              # starts on port 3000
curl http://localhost:3000/api/healthcheck # { "status": "ok" }
# Visual: open browser → layout shell renders with sidebar + main area
# Visual: navigate to /nonexistent → 404 handled gracefully
kill %1                                     # stop dev server
cd frontend && npm test -- --run            # all tests pass
cd frontend && npm run typecheck             # exits 0
cd frontend && npm run lint                  # exits 0
cd frontend && npm run format:check          # exits 0
```

**Size:** XS | **Dependencies:** T1

## Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
