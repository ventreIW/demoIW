# s1.2 Scope — Frontend Skeleton

## In scope

- `src/app/layout.tsx` — root layout with `<html lang="es">`, Tailwind base styles
- `src/app/page.tsx` — root page renders `MainLayout` with placeholder content
- `src/app/healthcheck/route.ts` — Next.js route handler returning `{ status: "ok" }`
- `src/components/layout/MainLayout.tsx` — shell: sidebar (nav links placeholder) + main area
- `src/components/layout/Sidebar.tsx` — placeholder nav links (Operaciones, Ejecutivo, Escenarios)
- shadcn/ui init (slate base color): `Button`, `Card`, `Separator`, `Skeleton`
- `src/styles/globals.css` — Tailwind directives
- `public/` — favicon placeholder
- Vitest test: `src/app/healthcheck/route.test.ts`
- ADR-001: App Router vs Pages Router (written as part of this story)

## Out of scope

- Actual navigation routing (pages don't exist yet)
- i18n setup (B-15)
- PWA manifest (B-14)
- Authentication / role switching
- Any data fetching or API calls to backend
- Dark mode toggle (deferred — not in backlog)
