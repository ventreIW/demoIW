## User story

> As a developer, I want the repository root to have consistent tooling configuration and a CI workflow so that all team members work with the same quality constraints and every pull request is automatically verified.

---

## Acceptance criteria

- [ ] Root `.gitignore` covers Node modules, Python caches, `.env`, `__pycache__`, `.mypy_cache`, `.next`, `dist`
- [ ] `frontend/` contains `package.json`, `tsconfig.json` (strict), `.eslintrc.json`, `prettier.config.js`, `vitest.config.ts`
- [ ] `backend/` contains `pyproject.toml` with `[tool.ruff]`, `[tool.mypy]`, `[tool.pytest.ini_options]` sections
- [ ] `backend/requirements.txt` and `requirements-dev.txt` are committed
- [ ] `.github/workflows/ci.yml` defines two parallel jobs: `frontend` and `backend`
- [ ] CI `frontend` job: `npm ci → typecheck → lint → test`
- [ ] CI `backend` job: `pip install → ruff check → mypy → pytest`
- [ ] CI triggers on `pull_request` to `main` and all `e*` branches
- [ ] `README.md` updated with local setup instructions for both apps
