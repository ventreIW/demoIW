# s1.1 Scope — Repository Scaffold & CI Pipeline

## In scope

- Root `.gitignore`
- `frontend/package.json` with scripts: `dev`, `build`, `typecheck`, `lint`, `test`
- `frontend/tsconfig.json` — `strict: true`, `paths` alias `@/*` → `src/*`
- `frontend/.eslintrc.json` — Next.js recommended + TypeScript rules
- `frontend/prettier.config.js` — single quotes, semi: false, tailwind plugin
- `frontend/vitest.config.ts` — jsdom environment, coverage thresholds
- `backend/pyproject.toml` — Ruff (line-length 100, select E,W,F,I), mypy (strict), pytest (testpaths = ["tests"])
- `backend/requirements.txt` — production deps (fastapi, uvicorn, sqlalchemy, asyncpg, alembic, pydantic-settings, httpx, structlog)
- `backend/requirements-dev.txt` — dev deps (pytest, pytest-asyncio, pytest-cov, mypy, ruff, httpx)
- `.github/workflows/ci.yml`
- `README.md` local dev setup section

## Out of scope

- Any application code (no `app/`, no `src/app/`)
- Docker / docker-compose
- Secrets management (`.env.example` comes in s1.3)
