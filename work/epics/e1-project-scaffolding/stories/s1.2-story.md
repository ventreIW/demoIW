# Story: Frontend Skeleton

## User Story
As a developer,
I want a frontend application with Next.js 15 App Router, TypeScript strict, Tailwind CSS and shadcn/ui, a basic layout shell, and a healthcheck API endpoint,
so that I can start building features with a solid foundation.

## Acceptance Criteria

### Scenario: Happy path
```gherkin
Given the development environment is set up
When I run the frontend dev server
Then the application starts without errors and displays the basic layout shell
And the healthcheck API endpoint returns { status: "ok" }
```

### Scenario: Edge case
```gherkin
Given the frontend is running
When I make a request to an undefined route
Then the application shows a 404 page (or handles gracefully)
```

## Examples (Specification by Example)

| Input | Action | Expected Output |
|-------|--------|-----------------|
| npm run dev | Start frontend dev server | Server starts on port 3000, no errors |
| curl http://localhost:3000/api/healthcheck | Call healthcheck endpoint | Returns { status: "ok" } |
| Visit root URL | Open browser | Shows layout shell with sidebar placeholder and main area |

## Notes
Based on epic design in work/epics/e1-project-scaffolding/design.md. Uses Next.js 15 App Router with TypeScript strict. Tailwind CSS and shadcn/ui for styling. Basic layout includes sidebar navigation placeholder and main area. Healthcheck API endpoint is implemented as a Next.js API route.
