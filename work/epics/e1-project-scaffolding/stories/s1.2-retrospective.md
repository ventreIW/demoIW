## What to improve

- **Design doc had wrong route path** ‚Äî `s1.2-design.md` specified `src/app/healthcheck/route.ts` but the acceptance criteria (story.md) expected `GET /api/healthcheck`. This mismatch should have been caught during design review.
- **shadcn v4 vs v3 surprise** ‚Äî the design assumed shadcn/ui v3 (`style=default, base=slate`) but `npx shadcn@latest` installed v4.11.0 with `base-nova` style and `@base-ui/react` instead of Radix. Version pinning should be in the design.
- **npm install on /mnt/c/ is unusable** ‚Äî the first `npm ci` attempt on the NTFS mount hung for 152s with zero output. This is a known WSL limitation but not documented in the project's README.

## Heutagogical Checkpoint

1. **What did I learn?**
   - Next.js App Router route handlers for `/api/*` URLs must live at `src/app/api/*/route.ts`, not `src/app/*/route.ts`. The URL path maps directly to the directory structure.
   - shadcn/ui v4 (4.11.0 as of June 2026) has breaking changes from v3: it uses `@base-ui/react` instead of Radix primitives, `base-nova` style replaces `default`, and `components.json` schema differs. The `npx shadcn@latest` version should be pinned.
   - WSL /mnt/c/ (NTFS-3g) is catastrophically slow for `npm ci` ‚Äî copying to native WSL filesystem (~/) is the workaround. This is a ~10x speedup.
   - ESLint flat config with `next/core-web-vitals` does NOT automatically ignore `.next/` in all cases ‚Äî an explicit `ignores: [".next/**"]` entry is needed.
   - Prettier also scans `.next/` unless `.prettierignore` is created. Both linter and formatter ignore files must be configured.

2. **What would I change about the process?**
   - Cross-reference route paths between design doc and story acceptance criteria before starting implementation. A 30-second check would have caught the `/api/healthcheck` vs `/healthcheck` mismatch.
   - Pin shadcn/ui version in the design: `npx shadcn@3` for v3 or document which v4 changes to expect.
   - Add a WSL setup section to the project README documenting the npm-on-native-fs workaround.

3. **Framework improvements?**
   - The `rai-story-design` skill for frontend stories should prompt: "Do route paths in the design match the acceptance criteria URLs?"
   - Design docs should include tool versions when they matter (shadcn/ui, Next.js major version, etc.).

4. **More capable of now?**
   - Next.js 15 App Router full bootstrap: route handlers, layouts, pages, Tailwind, shadcn/ui init.
   - Diagnosing and fixing WSL /mnt/c/ npm performance issues.
   - shadcn/ui v4 init with `@base-ui/react` dependency management.
