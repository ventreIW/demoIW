## User story

> As a developer, I want a working FastAPI backend that connects to PostgreSQL and exposes a health endpoint so that all backend stories have a deployable base application to build into.

---

## Acceptance criteria

- [ ] `cd backend && uvicorn app.main:app --reload` starts without errors
- [ ] `GET /health` returns `{ "status": "ok", "db": "connected" }` when PostgreSQL is reachable
- [ ] `GET /health` returns `{ "status": "degraded", "db": "unavailable" }` (503) when PostgreSQL is not reachable — no unhandled exception
- [ ] `alembic upgrade head` runs against a local PostgreSQL without errors (applies the init migration)
- [ ] `.env.example` is committed listing all required environment variables with placeholder values
- [ ] `pytest tests/ -v` finds and passes at least the health endpoint test
- [ ] `mypy app/` exits 0
- [ ] `ruff check .` exits 0
- [ ] OpenAPI docs available at `GET /docs` in development (`app_env=development`)
