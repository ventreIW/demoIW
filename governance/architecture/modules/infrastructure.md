## Configuration (Pydantic Settings)

All runtime config is read from environment variables. No secrets in code or committed files.

```python
class Settings(BaseSettings):
    # Database
    database_url: str                  # e.g. postgresql+asyncpg://...

    # OpenRouter
    openrouter_api_key: str
    openrouter_base_url: str = "https://openrouter.ai/api/v1"

    # Model selection per use case (GR-AI-002)
    model_data_enrichment: str = "mistralai/mistral-7b-instruct"
    model_communications: str = "anthropic/claude-3-haiku"
    model_nl_query: str = "mistralai/mistral-7b-instruct"

    # App
    app_env: str = "development"
    log_level: str = "INFO"

    class Config:
        env_file = ".env"
```

A `.env.example` is committed; `.env` is gitignored.

## Database

- **Async engine:** `asyncpg` driver via `SQLAlchemy 2.0` async API.
- **Session scope:** per-request session via FastAPI `Depends`.
- **Migrations:** Alembic with autogenerate from ORM models. Every schema change requires a migration script — no `create_all()` in production code.

---

## Migrations workflow

```bash
# Create a new migration after changing ORM models
cd backend && alembic revision --autogenerate -m "describe change"

# Apply migrations
cd backend && alembic upgrade head

# Rollback one step
cd backend && alembic downgrade -1
```

## Logging

- Structured JSON logs via `structlog`.
- Log fields: `timestamp`, `level`, `event`, `module`, `request_id` (injected by FastAPI middleware).
- LLM calls additionally log: `model`, `prompt_tokens`, `completion_tokens`, `latency_ms`.
- No PII in logs (all data is synthetic, but log format should be clean regardless).
